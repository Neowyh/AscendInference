### ⚠️ 中等必要性（可选）

#### 5. 批量推理（batch inference）
**必要性**: ⭐⭐⭐ 视情况而定

**分析**:
- ✅ 理论上提升吞吐量
- ⚠️ **实际实现问题**: 当前实现仍然是循环单张推理，不是真正的 batch
- ⚠️ 需要模型支持动态 batch
- ⚠️ 增加代码复杂度（+130 行）

**问题**:
```python
# 当前实现：伪 batch
for i in range(batch_size):
    # 单张推理
    self.preprocess(single_data)
    self.execute()
    self.get_result()
```

**建议**: 
- 如果模型不支持动态 batch → **移除**
- 如果需要真正 batch → 需要重新设计（修改模型加载）
- **当前优先级：降低**

---

#### 6. 异步推理（asyncio）
**必要性**: ⭐⭐ 可选

**分析**:
- ✅ 理论上支持高并发
- ⚠️ **实际场景有限**: 
  - 推理是 CPU/GPU 密集型，不是 IO 密集型
  - asyncio 对 CPU 密集型帮助有限
  - 多线程已经足够（multithread 模式）
- ⚠️ 增加代码复杂度（280+ 行）
- ⚠️ 学习曲线陡峭

**问题**:
```python
# asyncio 对 CPU 密集型任务帮助有限
async def inference_image():
    # 实际还是在 ThreadPoolExecutor 中执行
    # 不是真正的异步 IO
```

**建议**: 
- 对于大多数场景 → **可以移除**
- 多线程模式已经足够
- **当前优先级：低**
