# 三分支合并计划

## 摘要

- 最终目标分支：`master`
- 审查范围：
  - `master`
  - `codex/worktree-bootstrap`
  - `codex/ascend-yolo-system`
- 推荐合并顺序：
  1. 吸收 `codex/worktree-bootstrap` 中的整理性改动
  2. 先稳定 `master` 基线
  3. 再加固并合并 `codex/ascend-yolo-system`

## 合并前检查清单

### 基线检查

- 确认目标工作树足够干净，适合继续审查和合并操作。
- 重新执行 `pytest tests -q`，并明确记录：
  - 环境相关失败
  - 可复现的代码级失败
- 将基线稳定问题与分支特有问题拆开处理。

### `codex/worktree-bootstrap`

- 确认 `.worktrees/` 需要作为团队统一忽略目录。
- 确认 `.worktrees/` 下不存在必须纳入版本控制的资产。

### `codex/ascend-yolo-system`

- 检查以下命令及其文档签名是否一致：
  - `model-bench`
  - `strategy-bench`
  - `extreme-bench`
- 确认以下能力存在可接受的验收覆盖：
  - 标准输入分层评测
  - 遥感与大图路线
  - 策略评测
  - 报告生成与归档布局
- 核对配置词汇是否一致：
  - `Config` 支持的分辨率
  - 校验器输出信息
  - 输入分层的运行时映射

## 合并顺序

### 阶段一：吸收 `codex/worktree-bootstrap`

推荐做法：

- 保留 `.worktrees/` 的忽略行为。
- 优先使用直接 cherry-pick 或等价的单改动合并。
- 忽略规则落地后，分支本身即可退役。

原因：

- 该分支没有独立的长期集成价值。
- `codex/ascend-yolo-system` 已经包含相同提交。

### 阶段二：稳定 `master`

在主功能分支合并前，需要先完成以下工作：

- 修复或隔离基线测试失败
- 记录 Windows 临时目录权限噪音
- 对齐 logger 初始化语义与测试预期
- 明确导入检查、ACL 检查与路径校验的错误优先级

原因：

- 如果基线本身不可靠，就无法区分后续失败到底来自旧问题还是新功能。

### 阶段三：加固 `codex/ascend-yolo-system`

合并前阻塞项：

- 修复命令与文档不一致
- 解决分辨率词汇不统一
- 为新增评测与报告能力建立验收矩阵
- 在已稳定的基线上重新完成分支验证

推荐合并方式：

- 通过聚焦单一主题的 PR 合并到 `master`
- 审查重点放在集成边界，而不是只看文件数量

## 预期冲突点

- `.gitignore`
  - 风险较低，含义清晰
- `README.md`
  - 命令文档已经发生漂移，较容易冲突
- `docs/03-设计与实现/实现说明.md`
  - 如果基线文档在稳定阶段同步更新，容易产生内容冲突
- `config/config.py` 与 `config/validator.py`
  - 由于分辨率和评测语义变化，属于中等风险区
- `benchmark/` 与 `main.py`
  - CLI 接口和报告路径集成处容易出现冲突

## 合并后必须执行的验证命令

最少执行：

```bash
pytest tests -q
python main.py --help
python main.py model-bench --help
python main.py strategy-bench --help
python main.py extreme-bench --help
python scripts/run_smoke_eval.py --mode standard
python scripts/run_smoke_eval.py --mode remote
python scripts/run_smoke_eval.py --mode strategy
```

如果具备 Ascend 硬件环境，应追加执行带 `--run` 的 smoke 命令，并检查生成的报告与归档结果。

## 回滚与拆分策略

- 如果功能分支合并后验证失败，优先在 PR 级别回退或暂停，不要用强制重置共享历史的方式处理。
- 如果分支规模过大，不适合一次性稳妥合并，可以拆成以下子主题：
  - 评测契约与注册表层
  - 报告标准化与归档层
  - smoke 工具与文档同步

## 推荐的提交与 PR 切分方式

推荐顺序：

1. `chore: ignore local worktrees`
2. `chore: stabilize baseline tests and logging semantics`
3. `docs: align CLI and evaluation documentation`
4. `feat: merge evaluation, registry, and reporting system`
5. `chore: add post-merge acceptance coverage and archive verification`

## 合并建议

- 保留 `codex/worktree-bootstrap` 的改动内容，不保留分支身份。
- `codex/ascend-yolo-system` 必须在基线稳定且功能加固后再合并。
